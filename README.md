# Техническое задание на проект "Консольный файловый менеджер на os и os.system"

## 1. Общая информация

**Название проекта:** Консольный файловый менеджер на `os` и `os.system`

**Цель проекта:** Сделать простую программу, которая показывает, как работать с файлами и папками через стандартные библиотеки Python — `os`, `os.system`, `pathlib`. Без лишних заморочек, чтобы было понятно, что каждая функция делает.

## 2. Описание задачи (что нужно сделать)

Написать консольный файловый менеджер, который умеет:
- показывать, где ты находишься (текущая папка)
- выводить список файлов и папок
- создавать и удалять папки
- удалять файлы
- запускать системные команды (проводник, терминал, очистку экрана)

Всё это должно работать через простое меню с цифрами. Пользователь нажимает цифру — программа делает действие и возвращается в меню.

## 3. Функциональные требования (что система должна уметь)

1. Программа должна показывать меню со списком команд и номерами от 1 до 6.
2. Должна быть команда для выхода (0).
3. Команда 1 — показать текущую рабочую папку.
4. Команда 2 — показать содержимое текущей папки (файлы и папки).
5. Команда 3 — создать новую папку (пользователь вводит имя).
6. Команда 4 — удалить папку (пользователь вводит имя).
7. Команда 5 — удалить файл (пользователь вводит имя).
8. Команда 6 — открыть системный файловый менеджер (Проводник на Windows, Finder на Mac, Nautilus/Dolphin на Linux).
9. Программа должна работать на Windows, Linux и Mac.
10. После выполнения команды программа должна ждать нажатия Enter, чтобы вернуться в меню.
11. Если пользователь ввёл неправильную цифру — программа должна сказать об этом и показать меню снова.
12. При удалении папки или файла программа должна проверять, существует ли он, и если нет — говорить об этом, а не падать с ошибкой.

## 4. Нефункциональные требования (ограничения)

1. Язык программирования: Python 3.10 или новее.
2. Можно использовать только стандартную библиотеку Python: `os`, `os.system`, `sys`, `pathlib`.
3. Никаких сторонних библиотек.
4. Код должен быть простым и понятным, чтобы даже новичок разобрался.
5. Все пути должны работать на разных ОС (не должно быть захардкоженных слешей в ту или иную сторону).
6. Программа не должна падать от любых действий пользователя (неправильное имя, несуществующий файл и т.д.).

## 5. Требования к интерфейсу

При запуске программы пользователь видит примерно такое:

```
=== Файловый менеджер ===
1. Показать текущую папку
2. Показать содержимое папки
3. Создать папку
4. Удалить папку
5. Удалить файл
6. Открыть файловый менеджер системы
0. Выход

```

После выбора команды программа делает своё дело и выводит результат. Например:

```
Текущая папка: C:\Users\Windows\Desktop\os.sys
Нажмите Enter, чтобы продолжить...
```

Пользователь жмёт Enter — и снова видит меню.

## 6. Входные и выходные данные

10. Если пользователь вводит букву вместо цифры — программа не падает.

## Как это работает

### Определение операционной системы

В начале программы проверяется, на какой ОС мы работаем:

```python
import os
import sys
from pathlib import Path

if sys.platform == "win32":
    # Windows
    CLEAR_CMD = "cls"
    FILE_MANAGER_CMD = "explorer ."
    TERMINAL_CMD = "start powershell"
elif sys.platform == "darwin":
    # macOS
    CLEAR_CMD = "clear"
    FILE_MANAGER_CMD = "open ."
    TERMINAL_CMD = "open -a Terminal ."
else:
    # Linux
    CLEAR_CMD = "clear"
    FILE_MANAGER_CMD = "xdg-open ."
    TERMINAL_CMD = "gnome-terminal"
```

Это нужно, чтобы команды вроде `cls` (Windows) и `clear` (Linux/Mac) работали как надо.

### Показ текущей папки

```python
def show_current_dir():
    current = Path.cwd()
    print(f"\nТекущая папка: {current}")
```

`Path.cwd()` даёт путь в правильном формате для любой ОС.

### Показ содержимого папки

```python
def list_directory():
    current = Path.cwd()
    print(f"\nСодержимое папки {current}:\n")
    
    for item in current.iterdir():
        if item.is_dir():
            print(f"{item.name}")
        else:
            print(f"{item.name}")
```

`iterdir()` показывает всё внутри папки. `is_dir()` проверяет, папка это или файл. Потом рисуем иконки для красоты.

### Создание папки

```python
def create_folder():
    name = input("Введите имя папки: ")
    if not name:
        print("Имя не может быть пустым")
        return
    
    new_path = Path.cwd() / name
    try:
        new_path.mkdir()
        print(f"Папка '{name}' создана")
    except FileExistsError:
        print(f"Папка '{name}' уже существует")
    except Exception as e:
        print(f"Ошибка: {e}")
```

`Path.cwd() / name` — безопасно склеивает путь со слешами. `mkdir()` создаёт папку. Если папка уже есть — ловим ошибку и говорим об этом.

### Удаление папки

```python
def delete_folder():
    name = input("Введите имя папки для удаления: ")
    target = Path.cwd() / name
    
    if not target.exists():
        print(f"Папка '{name}' не найдена")
        return
    
    try:
        target.rmdir()  # удаляет только пустую папку
        print(f"Папка '{name}' удалена")
    except OSError:
        print(f"Папка '{name}' не пуста. Удалите сначала всё внутри.")
```

`rmdir()` удаляет только пустые папки. Если внутри что-то есть — программа скажет об этом, а не упадёт.

### Удаление файла

```python
def delete_file():
    name = input("Введите имя файла для удаления: ")
    target = Path.cwd() / name
    
    if not target.exists():
        print(f"Файл '{name}' не найден")
        return
    
    if target.is_dir():
        print(f"'{name}' — это папка. Используйте удаление папок.")
        return
    
    target.unlink()
    print(f"Файл '{name}' удалён")
```

`unlink()` удаляет файл. Предварительно проверяем, что это точно файл, а не папка.

### Запуск файлового менеджера

```python
def open_file_manager():
    os.system(FILE_MANAGER_CMD)
    print("Файловый менеджер открыт")
```

`os.system()` запускает команду, которая была определена в зависимости от ОС.

### Главный цикл программы

```python
def main():
    while True:
        print_menu()
        choice = get_user_choice()
        
        if choice == 0:
            print("До свидания!")
            break
        elif choice == 1:
            show_current_dir()
        elif choice == 2:
            list_directory()
        elif choice == 3:
            create_folder()
        elif choice == 4:
            delete_folder()
        elif choice == 5:
            delete_file()
        elif choice == 6:
            open_file_manager()
        else:
            print("Неверный выбор. Попробуйте снова.")
        
        input("\nНажмите Enter, чтобы продолжить...")
        os.system(CLEAR_CMD)  # очищаем экран
```

Всё крутится в бесконечном цикле, пока пользователь не нажмёт 0. После каждого действия чистим экран, чтобы меню выглядело аккуратно.

## Какие библиотеки используются и зачем

| Библиотека | Что даёт | Где используется |
|------------|----------|------------------|
| `os` | `os.system()` для запуска команд, `os.getcwd()` для текущей папки | Открытие файлового менеджера, очистка экрана |
| `os` | `os.mkdir()`, `os.rmdir()`, `os.remove()` | Создание и удаление (но в коде через `pathlib`) |
| `sys` | `sys.platform` | Определяем Windows/Linux/Mac |
| `pathlib` | `Path.cwd()`, `Path.iterdir()`, `Path.exists()`, `Path.is_dir()`, `Path.mkdir()`, `Path.rmdir()`, `Path.unlink()` | Вся работа с путями и файловой системой |

## Почему pathlib лучше, чем просто os

Можно было бы писать `os.path.join()` и `os.listdir()`, но `pathlib` удобнее:

```python
# Старый способ через os
path = os.path.join(os.getcwd(), "folder", "file.txt")

# Новый способ через pathlib
path = Path.cwd() / "folder" / "file.txt"
```

Во втором варианте короче и понятнее, особенно если много вложенностей.

## Про ограничения, которые всплыли

### 1. Нельзя удалить непустую папку
`os.rmdir()` и `Path.rmdir()` удаляют только пустые папки. Это сделано специально, чтобы случайно не стереть важные данные. Если нужно удалить папку со всем содержимым — пришлось бы использовать `shutil.rmtree()`, но это уже другая библиотека.

### 2. Нет нормального показа скрытых файлов
На Linux папки и файлы, начинающиеся с точки (`.git`, `.config`), считаются скрытыми. `iterdir()` их показывает, но визуально они ничем не отличаются.

### 3. os.system() не даёт контроля
`os.system("explorer .")` просто запускает команду и ждёт её завершения. Нельзя проверить, открылось всё нормально или нет. Для простого учебного проекта это норм, но для серьёзного приложения лучше использовать `subprocess`.

### 4. Кодировка имён файлов
На Windows с русскими именами файлов `pathlib` справляется нормально. На старых Linux-системах с не UTF-8 локалью могут быть проблемы.

## Выводы

Библиотеки `os` и `pathlib` закрывают 90% бытовых задач по работе с файлами и папками. `os.system()` удобен для простых вещей вроде "открыть проводник", но для сложных вызовов лучше брать `subprocess`. Главный плюс всего этого — всё работает из коробки, не надо ничего ставить через pip. Проект получился простым, рабочим и понятным.
